## 🎉 **Tutorial Lengkap: Menambahkan Embedding ke Vector Database**

### **📋 Prerequisites**
1. **PostgreSQL dengan pgvector** sudah berjalan (container `chatbot_postgres`)
2. **OpenAI API Key** di `.env.local`
3. **Node.js** terinstall
4. **Dependencies** sudah terinstall (`npm install`)

---

### **🚀 4 Cara Menambahkan Data Embedding**

#### **1. Dari File Tunggal (.txt/.md)**
```bash
cd /media/adi/Data/Firebase/chat-bot/chatbot-bisnisdigital

# Format: node ingest-postgres.js --file <path> --source "<nama_source>"
node ingest-postgres.js --file artikel-saya.txt --source "Artikel Blog #1"
```

#### **2. Dari Folder (Multiple Files)**
```bash
# Buat folder dan isi dengan file .txt/.md
mkdir blog-posts
cp *.txt blog-posts/

# Ingest semua file di folder
node ingest-postgres.js --dir blog-posts --source "Koleksi Blog 2024"
```

#### **3. Dari URL**
```bash
# Ingest dari URL (harus accessible)
node ingest-postgres.js --url https://example.com/artikel.txt --source "Artikel Eksternal"
```

#### **4. Dengan Custom Chunk Size**
```bash
# Default 1000 chars, bisa customize
node ingest-postgres.js --file artikel-panjang.txt --source "Artikel Panjang" --chunk-size 800
```

### **🔧 Troubleshooting**

#### **Error: Cannot find package 'openai'**
✅ **Sudah diperbaiki** - Script menggunakan `@ai-sdk/openai`

#### **Error: getaddrinfo EAI_AGAIN postgres**
```bash
# Jalankan dengan host localhost untuk development
POSTGRES_HOST=localhost node ingest-postgres.js --file file.txt --source "Test"
```

#### **Error: OpenAI API Key missing**
Tambahkan ke `.env.local`:
```
OPENAI_API_KEY=sk-your-key-here
```

### **🗂️ Struktur Database**

```sql
-- Tabel utama untuk chunks dengan embedding
CREATE TABLE documents (
  id SERIAL PRIMARY KEY,
  content TEXT NOT NULL,           -- Teks chunk
  source VARCHAR(255) NOT NULL,    -- Sumber dokumen
  embedding vector(1536),          -- Vector embedding (1536 dimensi)
  chunk_index INTEGER,             -- Index chunk dalam dokumen
  metadata JSONB,                  -- Info tambahan (fileName, dll)
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Index untuk search cepat
CREATE INDEX idx_documents_embedding 
ON documents USING ivfflat (embedding vector_cosine_ops);
```

### **⚡ Performance Tips**

1. **Chunk Size**: 800-1200 karakter optimal untuk context
2. **Rate Limiting**: Script sudah include delay 500ms antar API call
3. **Batch Processing**: Untuk banyak file, gunakan `--dir`
4. **Re-ingestion**: Script otomatis hapus data lama dengan source sama
